List of Bugs:

1) There is no way for a Troll to pull a rock when it moves. There needs to be some UX allowing a troll to select a rock to pull with it, as it moves

2) There is no UX for a wizard to select a rock to levitate as it moves. There needs to be a mechanism for the wizard to select a rock, then move, allowing the rock to move in the same motion as the wizard, but only if there is nothing blocking the rock as it moves.

3) When a Troll threw a rock, it correctly stopped at another rock in its path and correctly killed (Splut) the dwarf in front of that rock, but then all troll pieces were taken off the board. This should not have happened.

4) When the window is at full width, and as the Battle log grows, the length of the page expands causing the entire board to move down. The battle log needs to be encased in its own scrollable div such that the length of the entire page does not grow.

5) The battle log indicates positions such at E8 or D3 but these letters/numbers are not indicated on the broad. There should be letters and numbers on the broad so users can trace positions from the battle log

6) The board is rotated 45 degrees, making it difficult to understand North, South, East and West. When the Troll was throwing a rock, we were asked which direction to throw it in, but it was not clear which way was South. We should rotate the board back 45 degrees so it shows squares instead of diamonds.
